前端快爆


  • Chrome 118 发布：
    • CSS 支持 @scope (selector1) to (selector2) 规则，它是 CSS Cascading and Inheritance Level 6 的草案内容，在不增加优先级（specificity）的前提下，可匹配内部元素的样式；
    • 同时，作为上一项的特殊场景，支持省略 selector 的 @scope 规则，允许为 <style> 的父容器设置隔离样式规则；
  • Chrome 117 发布：
    • 支持 CSS Transitions Level 2 中定义的 transition-behavior，搭配 @starting-style，用于设置进入和退出离散值时的平滑动画；
    • CSS 中 grid-template-columns 和 grid-template-rows 支持了 subgrid 值，此前 Firefox 79 已支持；
    • CSS 中支持 text-wrap: pretty 用于美化字体排印场景中多行字最后一行只有一个单词的场景；
    • ECMAScript 支持了分组方法 Object.groupBy、Map.groupBy；
  • Chrome 116 发布：
    • 支持了为任意节点开启画中画 API requestWindow()；
    • CSS 运动轨迹属性 offset-path 支持了极坐标系 ray()、基础图形如方形 xywh()和 rect()、圆形 circle()、椭圆 ellipse() 和多边形 polygon()；
  • Chrome 115 发布：
    • 支持了滚动动画 animation-timeline: scroll()，允许一个动画的过程由鼠标滚动控制，此前类似功能只能通过 JavaScript 完成；
    • 支持隐私沙盒 fencedframe，类似 iframe，在其中能开启提供基于兴趣的广告服务的 Protected Audience API 和访问未分区的跨网站数据的 Shared Storage。
点评：Chrome 118 开始每三周发布一个版本，功能升级将越来越碎片化；隐私沙盒又是一个以保隐私为由保营收的操作。


  • Safari 17.1 发布：
    • iPhone 中也支持了 Managed Media Source API，此功能用于替代 Media Source Extensions，允许在低抽象层级上更高效地控制流媒体视频，管理缓冲区级别、控制网络请求和切换媒体；
  • Safari 17.0 发布：
    • API 更新：
      • 支持了快速高效编解码的 JPEG XL 和 HEIC 图片格式；
      • 支持了 Storage API，定义了通用共享的存储系统，允许获取使用空间并管理其中内容；
      • Web Worker 可渲染的离屏画布 Offsceen Canvas 支持了 WebGL，此前只支持 2D 离屏画布；
    • 支持了 ECMAScript 的新特性：
      • 正则表达式允许重复命名捕获分组的出现，该项提案当前处于 Stage 3；
      • 正则表达式支持集合操作，形同 /[\p{Script=Khmer}&&[\p{Letter}]/v，该提案进入 Stage 4，将进入明年的 ECMAScript 规范；
      • 支持了集合操作方法，如交集 intersection()、并集 union()、差集 difference() 等方法，该项提案当前处于 Stage 3；
    • HTML 更新：
      • 支持搜索标签 <search>，提供正确的无障碍语义，并为其内容默认设置其可访问性；
      • 支持 popover 属性，原生支持了浮层类交互；
    • CSS 更新：
      • 支持无前缀的连字符处理 hyphens 和 hyphenate-character，此前通过 -webkit- 前缀进行支持；
      • 支持了多图片序列 image-set()，可用于自适应浏览器支持的不同类型或不同分辨率的图片；
      • 支持切换为中日韩文的全角 text-transform: full-width 和切换为全宽片假名 text-transform: full-size-kana。
      • 允许为文字设置字号调整参考 font-size-adjust，允许字体匹配回退（fallback）时多字体视觉大小保持一致，它的默认值为 none，可设置为按字体调整 from-font，或按二值进行匹配，其中包括按大写高度设置 cap-height、按“0”的宽设置 ch-width、按“x”的高设置 ex-height、按“水”的宽设置和按“水”的高设置 ic-height。



  • Node.js 21 正式发布；
    • fetch 模块和 WebStreams 稳定状态，包括 WebStreams, FormData, Headers, Request, Response 和 fetch；
    • V8 升级到了 11.8，支持了分组方法 Object.groupBy、Map.groupBy，数组缓冲转移和解离系 API ArrayBuffer.prototype.transfer、ArrayBuffer.prototype.transferToFixedLength、ArrayBuffer.prototype.detached和 WebAssembly 中的 const 表达式扩展提案；
    • URL、fetch、streams、node:fs、HTTP 模块的性能有了提升；
    • 默认打开了 llhttp 9.1.2 的严格模式；
    • 此更新同时标识着 Node.js 20 进入了 LTS（长期支持）状态，直到 2024 年 10 月。


百宝箱


  • Yarn 发版 4.0：
    • 支持加固模式 enableHardenedMode，此模式下强制校验 lockfile 的解析范围与本地一致、metadata 远端和本地一致；
    • 默认可用 Node.js corepack 中设定的全局二进制包，同时要求 Node.js 升级到 18 以上；
    • 自动为 TypeScript 包增删相关 @types 依赖；
    • 性能表现提升 3 倍，几乎和 pnpm 相当。


  • Transformer.js 发版 2.7.0，支持文本转为语音特性。
点评：加载的模型文件还是出奇的大，加起来 500M，这样的限制下应用场景暂时还是比较难找的


  • Viselect 是一个允许拖拽多选的 UI 库。
点评：类似的交互 Fluent UI 也做了，可以参考一下


  • node-pureimage 是一个无原生代码依赖的 Node.js Canvas 绘制包，相比于传统包如基于 Cairo 的 node-canvas，可在树莓派等地方运行，代价是部分功能未实现。


  • Ruffle 是一个 Rust 实现的 Flash 播放器。
